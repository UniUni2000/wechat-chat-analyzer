# 微信聊天记录深度分析 Skill

基于 [wechat-decrypt](https://github.com/ylytdeng/wechat-decrypt) 解密后的微信本地数据库，对1对1单聊进行深度分析。

**架构**: Python 脚本负责数据计算（统计指标、图表、导出），LLM 负责深度分析（关系洞察、情感解读、性格画像、追求建议、文艺金句）。

## 数据准备（平台差异）

- **Windows**: 保持微信登录，以管理员权限启动 agent，脚本自动解密
- **macOS**: 需手动准备解密数据（详见 `references/platform_setup.md`）
- **Linux**: 需先在其他平台解密，将 `decrypted` 目录复制到工作路径

依赖由脚本自动检查和安装，无需手动处理。

## 分析流程

### Step 1: 提取参数并立即执行

遵循"最小干预原则"——**绝不向用户确认参数**。用户提到想分析聊天即立刻执行：

- 提取**好友标识**：用户提到的好友名字或上下文中显而易见的称呼
- 设定**自己的称呼**：默认 "我"

### Step 2: 静默执行脚本

**立刻使用终端工具执行，不要回复确认套话，不要让用户自己敲命令。**

```bash
python {SKILL_DIR}/scripts/wechat_analyzer.py \
  --friend {好友昵称} \
  --my-name {自己的称呼}
```

> 将 `{SKILL_DIR}` 替换为此技能根目录的绝对路径。不要用引号包裹路径和参数。

**脚本输出**:
- 统计数据：在 stdout 的 `LLM_ANALYSIS_DATA_START` / `LLM_ANALYSIS_DATA_END` 标记之间
- 图表 (PNG) 和导出文件 (TXT/CSV/HTML) 保存到输出目录
- 完整报告：`analysis_report.json`

### Step 3: 错误处理

脚本执行后检查退出状态：

- **退出码非 0**：检查错误信息，参考 `references/troubleshooting.md` 中的常见原因处理
- **提示 "未找到 decrypted 目录"**：引导用户按 `references/platform_setup.md` 准备解密数据
- **提示 "未找到聊天记录"**：确认好友名称是否正确，建议用户检查备注名或 wxid
- **依赖安装失败**：提示用户手动运行 `pip install -r requirements.txt`

### Step 4: 生成深度分析报告

基于脚本输出数据和采样对话生成完整分析报告。

- **报告模板**：参照 `references/report_template.md` 的格式
- **分析指南**：参照 `references/analysis_prompt_guide.md` 获取金句风格、性格画像、好感度走势等分析方法

如需更详细数据（如完整词频列表），读取输出目录中的 `analysis_report.json`。

### Step 5: 交互追问

报告生成后，提示用户可以继续追问（如分析某月变化、解读特定对话、生成表白文案等）。追问时读取 `analysis_report.json` 中的完整数据来回答。

## 关键注意事项

1. **隐私至上**: 报告中不要泄露 wxid 等敏感信息
2. **语气拿捏**: 专业有深度，但有温度、有网感，避免学术腔
3. **金句质量**: "一语胜千言"必须精炼、有画面感、能打动人心
4. **追求建议**: 具体可执行；单方面付出的关系要诚实但温柔地指出
5. **数据驱动**: 所有分析基于真实数据，不编造统计结论
6. **vendor 目录**: 不要删除或修改 `scripts/vendor/` 中的文件，解密功能依赖此目录
